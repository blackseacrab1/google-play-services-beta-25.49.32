# План тестирования Google Play Services 25.49.32

## Общие правила
- Тестировать минимум 48–72 часа для выявления фоновых проблем.
- Фиксировать всё: скриншоты, логи (adb logcat или встроенные инструменты), время событий.
- Использовать чистую среду (без root, без отключённых системных служб).
- Указать устройство: модель, версия Android, архитектура.

## 1. Push-уведомления (приоритет: высокий)
- Gmail: отправить письмо → задержка <10 сек
- YouTube: подписка + колокольчик, лайк/коммент → уведомление
- Google Messages (RCS/SMS)
- Google Maps (уведомления о трафике/навигации)
- Calendar, Photos, Keep, Chat
- Проверить в Doze-режиме и после авиарежима

Фиксировать: задержки, пропуски, дубли.

## 2. Синхронизация аккаунта
- Добавить/удалить второй аккаунт
- Проверить Contacts, Calendar, Drive, Photos, Keep
- Принудительная синхронизация в Настройках → Аккаунты
- После перезагрузки и авиарежима (5–10 мин)

Фиксировать: ошибки, бесконечный спиннер, потеря данных.

## 3. Фоновые службы и местоположение
- Location History в Maps
- Find My Device (android.com/find)
- Nearby/Quick Share
- Google Assistant

## 4. Расход батареи и трафика
- Мониторить 48+ часов (AccuBattery/GSam или встроенная статистика)
- Норма: <3–5% батареи в сутки на "Сервисы Google Play" в фоне
- Трафик: <50 МБ/сутки в фоне (без backup)

Фиксировать: скриншоты статистики батареи/трафика.

## 5. Стабильность и краши
- 5–10 перезагрузок → сервисы поднимаются без ошибок
- Авиарежим → выход → восстановление push/sync
- Принудительная остановка приложения → запуск
- Отключение/включение (осторожно!)

## 6. Дополнительные тесты
- Play Integrity API (приложение "Play Integrity API Checker")
- Google Wallet/Pay (если используете)
- Cast/Chromecast
- Автозаполнение паролей
- Работа с VPN/медленным интернетом

## Как фиксировать результаты
- Баги → папка bug_reports (описание + скриншоты + логи)
- Логи → adb logcat -d > log.txt (или через настройки)
- Скриншоты → папка screenshots

Результаты обновляются по мере тестирования.
